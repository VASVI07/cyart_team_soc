Threat Hunting Practice

System/VM Used: Ubuntu

Objective:
To investigate suspicious activity by reviewing active network connections on the Ubuntu VM.

Tool Used:
- Ubuntu Terminal
- ss command

Hypothesis:
Suspicious outbound network connections may be running from the Ubuntu VM.

What I Did:
1. Opened the Ubuntu VM terminal.
2. Ran the command ss -tunap to review active TCP and UDP connections.
3. Checked the output for established connections, unusual ports, and unknown external IP addresses.

What I Observed:
- Multiple listening services were visible on the system.
- Most established connections shown were local loopback connections using 127.0.0.1.
- No clearly suspicious remote external IP was confirmed from this check.
- This step helped verify currently active local and network-facing connections on the VM.
- Process review showed active services such as gnome-shell, redis-server, gvmd, ospd-openvas, and VBoxClient, which matched the lab and VM environment.


MITRE ATT&CK Mapping:
Initial network connection review for possible suspicious activity. No confirmed malicious technique was identified from this step alone.
