Evidence Analysis Practice

System/VM Used: Ubuntu

Objective:
To collect and review network connection evidence from the Ubuntu VM.

Tool Used:
- Ubuntu Terminal
- ss command

Evidence Type:
Network connection snapshot saved as a text file

What I Did:
1. Opened the Ubuntu VM terminal.
2. Ran the command ss -tunap > network-evidence.txt.
3. Saved the active network connection output into a text file for later evidence review.

What I Observed:
- The command created a file named network-evidence.txt successfully.
- The file stored active TCP and UDP connection details from the Ubuntu VM at the time of collection.
- This evidence can be used later to review currently active connections and check for unusual network activity.

Chain of Custody / Evidence Note:
The evidence file was collected manually from the Ubuntu VM terminal during lab activity and saved locally as network-evidence.txt.

Action Taken:
Collected and preserved the network connection output as a text evidence file for analysis.
